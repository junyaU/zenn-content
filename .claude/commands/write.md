# Write フェーズ

# ユーザの入力
$ARGUMENTS

## 目的
plan.mdに基づき記事を執筆する。技術的に正確で、読者が理解しやすい記事を作成する。

## 注意事項
- 常にultrathinkでしっかりと考えて作業を行うこと
- 技術的な正確性を最優先
- 読者の理解を助ける具体例を豊富に
- 既存の記事スタイルに合わせる
- 執筆後は `npx zenn preview` でプレビュー確認

## 必要な入力ファイル
- `docs/plan/plan_{TIMESTAMP}.md` - 執筆計画書
- 既存の記事（articles/）- スタイル参照用

## タスクに含まれるべきTODO
1. ユーザの指示を理解し、執筆開始をコンソールで通知
2. 最新の `docs/plan/plan_{TIMESTAMP}.md` ファイルを読み込み、執筆計画を確認
3. 新しい記事ファイルを作成（`npx zenn new:article`）
4. プランに従って記事を段階的に執筆
   - 導入部分
   - 各技術セクション
   - コード例
   - 図解
   - まとめ
5. 執筆後のチェック
   - 技術的な正確性
   - コード例の動作確認
   - 誤字脱字
6. プレビューで確認: `npx zenn preview`
7. 執筆内容の詳細を `docs/write/write_{TIMESTAMP}.md` に記録

## 執筆ガイドライン
### 文章スタイル
- **一人称**: 「私」または省略
- **文体**: である調
- **専門用語**: 初出時に説明を加える

### コード例
- **言語指定**: コードブロックに言語を明示
- **コメント**: 重要な部分にのみ日本語コメント
- **動作確認**: 実際に動作することを確認

### 図解
- **Mermaid**: フロー図やシーケンス図
- **ASCII**: シンプルな構造図

### Zenn記法
- **メッセージ**: `:::message`
- **アコーディオン**: `:::details`
- **数式**: KaTeX記法

## 執筆記録ドキュメントテンプレート
```markdown
# Writing Details: [記事タイトル]

## 執筆サマリー
[執筆した記事の概要]

## プラン参照
- Plan file: docs/plan/plan_{TIMESTAMP}.md

## 執筆詳細
### 導入
- 内容: [導入で書いた内容]
- 文字数: [文字数]

### セクション1: [セクション名]
- 内容: [書いた内容]
- コード例: [含めたコード例]
- 図解: [含めた図解]

### セクション2: [セクション名]
- 内容: [書いた内容]
- コード例: [含めたコード例]
- 図解: [含めた図解]

### まとめ
- 内容: [まとめで書いた内容]

## 品質チェック
- [ ] 技術的に正確
- [ ] コード例が動作する
- [ ] 誤字脱字なし
- [ ] プレビューで表示確認

## 次のステップ
- [ ] レビュー依頼
- [ ] 公開設定（published: true）
```

## 出力ファイル
- `docs/write/write_{TIMESTAMP}.md` - 執筆詳細記録

## 最終出力形式
- 必ず以下の形式で出力を行ってください

### 執筆完了の場合
status: SUCCESS
next: REVIEW
details: "記事執筆完了。write_{TIMESTAMP}.mdに詳細記録。プレビュー確認済み。"

### 追加執筆が必要な場合
status: NEED_MORE
next: WRITE
details: "追加執筆が必要。write_{TIMESTAMP}.mdに詳細記録。タスク継続。"

### プラン見直しが必要な場合
status: NEED_REPLAN
next: PLAN
details: "構成変更が必要。write_{TIMESTAMP}.mdに詳細記録。プランフェーズに戻る。"
