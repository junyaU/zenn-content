# Analyze Phase

# ユーザの入力
$ARGUMENTS

## 目的
記事のテーマに関する技術的調査を行い、記事の構成と内容を決定する。

## 注意事項
- 全ての処理においてultrathinkでしっかりと考えて作業を行うこと
- 読者の理解レベルを考慮すること
- 既存の記事（articles/）を参照し、執筆スタイルを把握すること

## タスクに含まれるべきTODO
1. 調査対象・スコープを明確化
   - 記事のテーマと目的
   - 対象読者（初心者/中級者/上級者）
   - 技術的な深さのレベル
2. 関連情報を収集し、体系的に分析
   - 公式ドキュメント
   - 技術仕様
   - 参考記事・書籍
3. 技術的な正確性を確認
   - 最新の情報かどうか
   - 実際に動作するかどうか
   - エッジケースの考慮
4. 記事の構成案を作成
   - 導入（なぜこのテーマが重要か）
   - 本文（技術的な解説）
   - まとめ（要点の整理）
5. 調査結果を文書化し、`docs/analysis/analyze_{TIMESTAMP}.md`に保存

## 調査ポイント
1. **読者視点の確認**
   - 前提知識は何が必要か
   - どこでつまずきやすいか
   - 具体例は何が適切か

2. **既存記事との整合性**
   - 用語の統一
   - 説明の深さ
   - 図解の有無

3. **実用性**
   - コード例は動作するか
   - 再現性はあるか

## 出力ファイル
- `docs/analysis/analyze_{TIMESTAMP}.md`

## ドキュメントテンプレート
```markdown
# Analysis Report: [記事テーマ]

## 概要
[記事の目的と対象読者]

## 調査範囲
- テーマ: [例: VirtIOネットワークドライバ]
- 対象読者: [例: OS開発に興味がある中級者]
- 前提知識: [必要な前提知識]

## 技術的詳細
### 主要概念
[解説すべき主要な概念]

### 難しいポイント
[読者がつまずきやすい点]

### コード例の方針
[どのようなコード例を含めるか]

## 記事構成案
### 導入
[導入部分の内容]

### 本文
1. [セクション1]
2. [セクション2]
3. [セクション3]

### まとめ
[まとめ部分の内容]

## 参考資料
- [資料1]
- [資料2]

## 次のステップ
[PLANフェーズでの推奨事項]
```

## 最終出力形式
- 必ず以下の形式で出力を行ってください

### 調査完了の場合

next: PLAN
